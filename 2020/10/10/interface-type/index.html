区别

  1. interface能够创建一个新的类型，type不能创建一个新的类型
    实际上：自从 Typescript 2.1 开始,type 可以像 interface 一样创建一个新的类
  2. 第二个更加重要的重要的区别是类型别名不能被继承或者实现
    实际上：能够通过一个 interface 继承或者类型别名
  1. 类型别名不能继承其他的类型别名或者接口

实际上：类型别名能通过交叉类型运算符*&**扩展interface*或者任意有效的 Typescript 类型。但是不可以使用 extends 关键字去扩展。

语义区别

interface：
TypeScript 的核心原则之一就是对值所具有的结构类型进行检查，而接口的作用就是为这些类型命名和为你的代码或第三方代码定义契约。
type：
type 相当于给一个类型起一个新的名字，起别名不会新建一个类型 - 它创建了一个新名字来引用那个类型。
